#### 什么是 IoC ？

简单来说，IoC 是反转控制，类似于好莱坞原则，主要有依赖查找和依赖注入实现。

#### 依赖查找和依赖注入的区别？

依赖查找是主动或手动的依赖查找方式，通常需要依赖容器或标准 API 实现。而依赖注入则是手动或自动依赖绑定的方式，无需依赖特定的容器和 API。

#### Spring 作为 IoC 容器有什么优势？

- 典型的 IoC 管理，依赖查找和依赖注入
- AOP 抽象
- 事务抽象
- 事务机制
- SPI 扩展
- 强大的第三方整合
- 易测试性
- 更好的面向对象

#### 轻量级容器有哪些特征？

- 能够管理应用代码；
- 能够快速启动；
- 容器不需要特殊的配置；
- 容器能够达到比较轻量级的内存占用以及最小化的 API 依赖；
